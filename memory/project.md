# 项目记忆（按需维护）

## 当前重点项目
- **项目名称**：工作数字助手自主进化系统
- **技术栈**：OpenClaw + Python/Bash + Cron + Git + Markdown
- **仓库地址**：/root/.openclaw/workspace
- **目标**：构建一个安全可控、可自我进化的工作数字助手系统

## 已知问题与解法
### 2026-03-09 AGENTS.md 身份分离架构变更
- **问题**：AGENTS.md 包含太多内容，角色定位与复杂规则混合
- **解法**：引入 IDENTITY.md 作为单独的身份配置文件
  1. **分离身份**：将"我是谁、性格、工作原则"移至 IDENTITY.md
  2. **优先加载**：IDENTITY.md 优先级最高，快速调整性格
  3. **简化架构**：AGENTS.md 专注于安全规则和进化机制
- **实现**：使用 Python 脚本执行 AGENTS.md 更新和备份

### 2026-03-09 AGENTS.md 绝对禁止清单扩充
- **问题**：身份分离后需要防止进化机制意外修改 IDENTITY.md
- **解法**：在 AGENTS.md 的绝对禁止清单中明确禁止自动修改 IDENTITY.md
- **规则内容**：
  ```
  - 通过进化机制自动修改 IDENTITY.md 的任何内容
  ```
- **重要性**：确保身份文件只能由人工修改，维护核心身份稳定性和一致性
- **安全意义**：这是一条不可逾越的红线，即使进化机制达到 Level 3 条件也必须遵守

## 重要约束
- **代码仓库权限**：使用 git 管理所有配置文件变更
- **不可触碰的目录/服务**：
  - 系统目录：/etc、/var、/usr
  - 凭证文件：git凭据、VPN配置、密钥文件
  - 生产环境：未经确认不能修改生产脚本
- **不可自动修改的文件**：
  - **IDENTITY.md**（最高优先级，禁止进化机制自动修改）
  - AGENTS.md（需要用户明确确认）
  - memory/core.md（需要用户明确确认）
  - skills/ 下的文件（需要用户明确确认）
